# Touch Smash — P2P パーティーテニス

Three.js による 3D スタジアム（ナイトセッション）と、WebRTC (PeerJS) による
サーバーレス P2P 対戦を備えたパーティーテニスです。対戦でポイントを貯めると、
**罰ゲーム/ご褒美ルーレットの当選確率が変わります**。

## 遊び方

**公開版（おすすめ）**: main に push すると GitHub Actions が自動で GitHub Pages に
デプロイします（`.github/workflows/deploy.yml`）。公開 URL をスマホで開けば、
**離れた場所の友達ともルームコードだけで対戦できます**（同じ Wi-Fi は不要）。

```bash
npm install
npm run dev        # 開発サーバー（LAN 公開: スマホから http://<PC-IP>:5173）
```

| モード | 手順 |
|---|---|
| ひとりで練習 | メニューから「ひとりで練習（AI対戦）」（ポイントは貯まりません） |
| パーティー | 1人が「パーティールームを作る」→ 5文字コードを共有 → 最大8人まで「ルームに参加」 |

P2P はインターネット経由で接続できます（シグナリングに PeerJS の無料クラウドを使用。
以降のゲーム通信は WebRTC DataChannel で直接 P2P）。

### フレンド機能（オプション）

自分の Firebase プロジェクト（無料）をつなぐと、ホームの「👥 フレンド」で
フレンドコードによる登録・オンライン表示・**友達がいる部屋へのワンタップ参加**が
使えます。セットアップ手順は [FIREBASE_SETUP.md](FIREBASE_SETUP.md) を参照。
未設定でもゲーム本体はすべて動きます。

### パーティーの流れ（勝ち残り制）

0. ルーム作成時に**ポイント共有グループ**を選択/新規作成。ポイントはグループごとに
   独立した台帳として永続保存され、同じグループで集まれば何度でも引き継がれる。
   ホストはロビーの ± ボタンで各メンバーのポイントをいつでも動的に調整できる
1. ロビーに集合。ホストが「次の試合を開始」（先取ゲーム数は 1/2/4 から選択）
2. 2人が対戦、ほかのメンバーは**観戦モード**（サイドスタンド視点）で生観戦
3. 勝者は +100pt・王冠つきで居残り、敗者は取ったゲーム × 20pt を獲得して列の最後尾へ
4. ポイントは**ホスト端末に過去データとして永続保存**（名前ごとの累計・勝敗）
5. ロビーで「💥 罰ゲームルーレット」「🎁 ご褒美ルーレット」（メンバー2人以上で使用可）
   - 罰ゲーム: ポイントが多い人ほど**当たりにくい**
   - ご褒美: ポイントが多い人ほど**当たりやすい**
   - 抽選はホスト権威で 1 回だけ行い、全員の画面で同じ結果に収束する回転演出

### 操作

- **PC**: 移動 = WASD / 矢印キー、ショット = スペース (Z)、ロブ = X (Shift)
- **スマホ/タブレット**（横持ち）: 左半分タッチでバーチャルスティック移動、右下の「ショット」「ロブ」ボタン
- 打つ瞬間にスティック/キーを左右に倒すとコースを打ち分けられます
- 高いボールをショットすると自動でスマッシュになります

### ルール

- 15 / 30 / 40 / デュース / アドバンテージの正式スコア、4ゲーム先取
- サーブは対角のサービスボックスに入らないとフォルト（2本失敗でダブルフォルト）
- ネットを越えない・アウト・ツーバウンドで相手のポイント

## アーキテクチャ

ホスト権威モデル: ボール物理とルール判定はルーム作成側（ホスト）だけが実行し、
30Hz のスナップショットを配信。クライアントは自機位置とスイング要求を送り、
受信間はローカル物理で補間します（`docs` は廃止し、コードのコメントが仕様です）。

```
src/
  config.ts            コート寸法（実寸）・物理・ゲーム定数
  sim/physics.ts       ボール物理（重力/バウンド/ネット/ドラッグ）と弾道ソルバ
  sim/score.ts         テニススコア進行
  sim/HostSim.ts       ホスト権威シミュレーション（サーブ/ラリー/判定）
  sim/ai.ts            練習モードの AI
  net/protocol.ts      P2P メッセージ定義（ロビー/試合/スナップショット/ルーレット）
  net/peer.ts          PeerJS ラッパー（ホスト=スター型で複数接続）
  party/party.ts       パーティー進行（勝ち残りローテ・ポイント付与・抽選）
  party/storage.ts     戦績の永続化（localStorage）
  view/Stadium.ts      3D スタジアム（コート/ネット/観客/照明/広告）
  view/PlayerView.ts   選手モデルとプロシージャルアニメ（フォア/バック）
  view/BallView.ts     ボール/軌道トレイル/バウンド演出
  game/Game.ts         ゲームループ・役割（対戦者/観戦者/権威）・カメラ
  ui/controls.ts       キーボード + タッチ入力
  ui/hud.ts            スコアボード/メッセージ/リザルト
  ui/roulette.ts       ルーレット演出（Canvas）
  core/audio.ts        WebAudio 効果音合成
  main.ts              メニューとモード起動
```
