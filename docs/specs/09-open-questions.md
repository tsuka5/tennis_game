# 09. 前提・確定事項・既定値

確定済みの主要設計判断は [README](./README.md) の Decision Log（D-1〜D-7）を参照。
本章は「仮置きした既定値」と、かつての未決事項（Q-1〜Q-10）の **確定結果** を示す。
**Q-1〜Q-10 はすべて確定済み**。以後の実装はこれを正とする（変更時は本章とDecision Logを更新）。

## 9.2 確定事項（旧 Open Questions）

### Q-1 難NAT環境の保証範囲 → 確定: **既定は非保証。TURNは後付け可能な設計**
- 同一Wi-Fi/近接利用（A-1）を前提とし、**STUNのみ**で接続する。
- STUNサーバは可用性のため**複数指定**する（例: `stun:stun.l.google.com:19302` ほか）（MUST）。
- 異なるモバイル回線同士など接続失敗時に備え、`GameConfig.net` に **TURNサーバ設定の差し込み口（iceServers配列）を用意**しておく（MUST）。既定は空（TURNなし）。
- TURNが必要になった場合は無料/従量のTURN（例: 自前coturn、またはマネージド）を後から追加するのみで動く構成とする。

### Q-2 シグナリングのホスティング先 → 確定: **Cloudflare Workers + Durable Objects**（D-5）
- ルームを **Durable Object** インスタンスとして表現し、`roomId` でルーティング。WebSocket接続をルーム単位で集約・中継する。
- 中継するのは [01](./01-architecture.md) 1.3 のシグナリングメッセージのみ。ゲーム状態は持たない（MUST NOT）。
- 無料枠で運用、コールドスタートによる初回接続遅延が小さく、常時スリープしない点を採用理由とする。
- ローカル開発は `wrangler dev`（Miniflare）で行い、本番デプロイは `wrangler deploy`（SHOULD）。
- クライアントの`wss://`接続先はビルド時環境変数で切替可能にする（MUST）。

### Q-3 ダブルスのボール権限「代表決定」 → 確定: **最近接優先 + 連続ヒット禁止 + 僅差はホスト裁定**
- 自チーム半面にボールがある間、**ボールに最も近い味方**が代表（ボール権限）となる（MUST）。
- 直前に打った本人は、相手から返ってくるまで連続して代表ヒットできない（自チーム内で交互、二度打ち防止）（SHOULD）。
- 2人の距離差が `DOUBLES_TIEBREAK_PX`（既定 24 論理単位）以内の僅差で同時ヒット主張が起きた場合は**ホストが裁定**（早い`t`優先、[05](./05-netcode-sync.md) 5.4.3）（MUST）。

### Q-4 得点判定が曖昧な競合のホスト裁定 → 確定: **早い`t`優先、判定不能はレット（やり直し）**
- 同一ボールに複数の主張が来た場合、ホストは送信時刻 `t`（オフセット補正後）が最も早い報告を採用する（MUST）。
- どちらが正か判定できない（時刻差が誤差範囲・報告矛盾）場合は、そのポイントを**レット**として無得点でサーブやり直しにする（MUST）。
- レット発生は両者へ通知し、`rallyId` を進める（[06](./06-data-models.md)）。

### Q-5 キャラの可動範囲 → 確定: **自コート内のみ、ネットを越えない**
- 各プレイヤーは自コート（[02](./02-coordinate-and-camera.md) 自半面）の内側のみ移動可能（MUST）。
- ネット（`NET_Y`）手前に移動上限を設け、相手コートへは侵入できない（MUST）。
- コート横方向はサイドラインまで（規定マージン内）。可動矩形は `GameConfig` で定義する（SHOULD）。
- ダブルスでは2人が同一半面を共有し、相互の重なりは許容（押し合いなし）（MAY）。

### Q-6 スマッシュ発動条件 → 確定: **ボール高さ閾値 または 高速接近時にAで発動**
- Aボタンでのヒット時、次のいずれかを満たすと**スマッシュ**（`SMASH_SPEED`適用）になる（MUST）:
  - ボールの擬似高さ `height >= SMASH_HEIGHT`（既定 0.6、ロブ/高い球を叩く想定）、または
  - ボールの接近速度が `SMASH_INCOMING_SPEED`（既定 600 論理/秒）以上。
- それ以外のAヒットは通常ショット（`SHOT_SPEED`）。
- 上記閾値は `GameConfig.shot` に追加する（[06](./06-data-models.md)）。

### Q-7 サウンド・演出の作り込み → 確定: **最小限**（D-7）
- 効果音はヒット音・得点音のみ（MUST）。BGM・複数SEは範囲外（MAY、後日拡張）。
- 演出は得点ハイライトとゲームセット画面の簡素な表示のみ（MUST）。
- iOS Safari の自動再生制限のため、**最初のユーザー操作（タップ）でAudioContextを解放**する（MUST、9.3参照）。

### Q-8 再接続時の状態復元 → 確定: **best-effort（保証しない）**
- 一時切断後、同 `roomId` への再参加を一定時間（`REJOIN_GRACE_MS`、既定 10000ms）受け付ける（SHOULD）。
- 復帰時はホストが現在の `MatchState` を送って同期を回復させる（SHOULD）。ボール状態の厳密な巻き戻しは行わず、進行中ポイントはレットにしてよい（MAY）。
- 猶予時間を超えた切断は離脱確定（[05](./05-netcode-sync.md) 5.6）。

### Q-9 対応ブラウザ/OS最低版 → 確定: **iOS Safari 15+ / Android Chrome 90+**（D-6）
- 上記2系統を一級サポート（MUST）。それ以外（PC含む）はデバッグ用途のbest-effort。
- iOS Safari 固有の制約を仕様に織り込む（9.3）:
  - **画面向きロック非対応**: `screen.orientation.lock()` はiOS Safariで使えないため、横画面強制は**CSS/オーバーレイ案内方式**で実現する（MUST、[03](./03-controls-ux.md) 3.4）。
  - **音声の自動再生制限**: 初回タップでAudioContext解放（Q-7）。
  - **Wake Lock**: 対応状況に差があるためフォールバック必須（[03](./03-controls-ux.md) 3.4）。
- 起動時に最低要件を満たさないブラウザには非対応メッセージを表示する（SHOULD）。

### Q-10 アナリティクス/ログ収集 → 確定: **なし（プライバシー優先）**
- 外部送信のアナリティクス・トラッキングは入れない（MUST NOT）。
- 開発時のデバッグログはローカルコンソール/開発ビルドに限定し、本番ビルドでは抑制する（SHOULD）。

## 9.3 確定により追加されるパラメータ

`GameConfig`（[06](./06-data-models.md) 6.6）に以下を追加する（MUST）。

```ts
// net に追加
iceServers: RTCIceServer[];     // 既定: 公開STUN複数。TURNは空（Q-1）
signalingUrl: string;           // wss:// 接続先（環境変数, Q-2）
rejoinGraceMs: number;          // 既定 10000（Q-8）

// shot に追加
SMASH_HEIGHT: number;           // 既定 0.6（Q-6）
SMASH_INCOMING_SPEED: number;   // 既定 600（Q-6）

// doubles（新設）
DOUBLES_TIEBREAK_PX: number;    // 既定 24（Q-3）
```

## 9.4 既知のリスクと緩和

| リスク | 影響 | 緩和策 |
|---|---|---|
| NAT越え失敗 | 接続できない | STUN複数指定、iceServersにTURN差し込み口を用意（Q-1） |
| iOS Safari制約 | 横画面/音が出ない | 向きはCSS+案内、音は初回タップ解放（Q-9） |
| 反転同期のズレ | 体験破綻 | 同期は論理座標のみ・逆変換テスト（AC-2.3）で担保 |
| ハンドオフ取りこぼし | ボール消失/二重化 | reliable送信＋hitId重複破棄（AC-4.2/4.3） |
| 同時ヒット競合 | スコア不整合 | ホスト裁定で単一収束、判定不能はレット（Q-4, AC-4.4） |
| ダブルスの二度打ち | 不公平/曖昧 | 最近接＋連続ヒット禁止＋僅差ホスト裁定（Q-3） |
| 実機性能不足 | カクつき | 描画/物理の軽量化、目標30fps下限（AC-6.1） |
| 無料枠の制限超過 | シグナリング不安定 | 規模監視、必要時に有料枠/VPSへ移行（[README](./README.md) 費用表） |
