# 00. 概要・目的・スコープ

## 0.1 プロダクト概要

**タッチ・スマッシュ** は、専用ゲームパッドやVR機器を一切使わず、各自のスマートフォンの**画面タッチ操作のみ**でリアルタイムに打ち合う、2Dトップダウン（真上見下ろし）視点のテニスゲームである。

WebブラウザのWebRTC（DataChannel）を用い、URL/QRコードを共有するだけでその場のプレイヤー同士が即座にP2P接続し、低遅延で対戦できる、Webアプリケーションとして構築する。

## 0.2 目的・成功指標（Goals / Success Metrics）

| ID | 目的 | 計測可能な成功指標 |
|---|---|---|
| G-1 | その場ですぐ繋がる | URL/QR共有から対戦開始まで **30秒以内**（同一Wi-Fi、ユーザー操作含む） |
| G-2 | ラグを感じさせない操作 | 自コートでのスイング入力からラケット反応まで **体感遅延0**（ローカル即時計算、[05](./05-netcode-sync.md)） |
| G-3 | 手元を見ずに操作できる | バーチャルジョイスティック＋大型透過ボタンにより、画面を注視せず操作可能（[03](./03-controls-ux.md)） |
| G-4 | 休み時間に1試合が終わる | 1試合の標準所要時間 **2〜4分**（3ポイント先取等のミニマル設定） |
| G-5 | 追加コスト最小 | 運用費 **実質¥0**（無料シグナリング + STUNのみ） |

## 0.3 想定ユーザー / 利用シーン

- 学校の休み時間、同じ場所（同一Wi-Fiまたは近接）にいる友人同士。
- 主にスマートフォン（横画面）。タブレットも可。
- PCブラウザは開発・デバッグ用途（フェーズ確認）として動作すればよい（MAY）。

## 0.4 スコープ

### やること（In Scope）

- スマホ横画面・タッチのみのトップダウン2Dテニス。
- 1人壁打ち（練習/通信テスト）、1vs1シングルス、2vs2ダブルス。
- WebRTC DataChannelによるP2P対戦と、ラグ隠蔽（自コート権限委譲）。
- クライアント側の画面180度反転（常に自分が手前）。
- ミニマルな試合設定（先取ポイント数）とスコア/ゲームセット演出。
- 無料シグナリングサーバ（最小WebSocket）。

### やらないこと（Out of Scope）

- ネイティブアプリ（iOS/Android）化。本書はWebのみ。
- アカウント制・永続ランキング・マッチメイキング（見知らぬ相手との自動対戦）。
- 5人以上、トーナメント、観戦モード。
- 課金、広告、サーバー側でのゲーム進行（ゲームロジックはクライアントP2Pで完結）。
- TURN必須となる難NAT環境の保証（任意対応、[01](./01-architecture.md) D-2）。
- ゲームパッド/キーボード操作の正式サポート（デバッグ用にMAY）。

## 0.5 用語定義

| 用語 | 定義 |
|---|---|
| ホスト（親機） | ルームを作成し、試合状態（スコア等）の権威を持つピア。プレイヤーA。 |
| クライアント（子機） | ルームに参加するピア。プレイヤーB（ダブルスではB/C/D）。 |
| ピア（Peer） | P2Pネットワーク上の各端末。 |
| 論理座標（Canonical Space） | 全ピアで共有する基準座標系。ホスト視点を基準とする（[02](./02-coordinate-and-camera.md)）。 |
| ローカル座標 | 各端末の画面に実際に描画する座標。自分が手前に来るよう変換される。 |
| 自コート | 論理コートを中央ネットで二分したうち、自分（自チーム）側の半面。 |
| ボール権限（Ball Authority） | ある瞬間にボールの物理シミュレーションを権威的に計算するピア。 |
| ハンドオフ（Handoff） | ボール権限が相手ピアへ移る同期イベント。 |
| ラリー | ボールが両者間を往復し続けている状態。 |
| ティック（Tick） | ゲームループ/同期送信の単位時刻。 |
