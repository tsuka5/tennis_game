# 03. タッチ操作・UI/UX 仕様

## 3.1 画面レイアウト（横画面固定）

```
+-------------------------------------------------------+
|  [自スコア]            [相手スコア]      [メニュー/退出]|
|                                                       |
|                      (コート描画)                      |
|     ( 左親指エリア = 画面左半分 )   ( 右親指エリア = 右半分 )|
|   +------------------------+    +---------------------+|
|   | 動的バーチャル          |    |  [A] ショット/スマッシュ ||
|   | ジョイスティック        |    |  (タップ/フリック)    ||
|   | (どこを触っても出現)    |    |  [B] ロブ            ||
|   +------------------------+    +---------------------+|
+-------------------------------------------------------+
```

- **MUST**: 画面を左右2分割し、左半分=移動、右半分=ショットの担当領域とする。
- **MUST**: 親指2本での同時操作（マルチタッチ）に対応する。左右の操作は独立して同時成立すること。
- **MUST**: 操作UIは透過表示とし、コート視認を妨げないこと。

## 3.2 移動操作（左親指エリア）— 動的バーチャルジョイスティック

要件:
- **MUST**: 画面左半分の**任意の位置**にタッチダウンすると、その接触点を中心としてバーチャルジョイスティックが出現する。固定配置にしない。
- **MUST**: 中心からの指のドラッグ方向（360度）に応じてキャラが移動する。方向は連続値（8方向固定ではない）。
- **SHOULD**: 中心からの距離で移動速度を段階制御（小さく倒す＝ゆっくり、最大倒し＝最高速）。デッドゾーンを設ける。
- **MUST**: 指を離すと移動入力は即座に0になり、ジョイスティックは消える。
- **MUST**: 入力は[02](./02-coordinate-and-camera.md) 2.5 に従い論理座標へ変換してから適用する（クライアントは左右上下反転）。

パラメータ（実装定数・既定値、調整可）:

| 定数 | 既定 | 説明 |
|---|---|---|
| `JOY_DEADZONE_PX` | 8 | これ未満のドラッグは無入力 |
| `JOY_MAX_RADIUS_PX` | 64 | この距離で最大速度 |
| `PLAYER_MAX_SPEED` | 320（論理単位/秒） | キャラ最高移動速度 |

## 3.3 ショット操作（右親指エリア）

右半分に**大型の透過ボタン2つ**を配置する。押し間違い防止のため十分な当たり判定サイズを確保（MUST）。

### 3.3.1 Aボタン：通常ショット / スマッシュ

- **MUST**: ボールが自キャラのヒット可能範囲（`HIT_RANGE`）内にある時にタップすると、ボールを相手コートへ打ち返す。
- **MUST**: ボールが範囲外の時のタップは「空振り」ではなく**無効入力**として扱い、失点にしない（範囲外で振っても可、当たらないだけ）。
- ヒット可能範囲外で意図的に振った場合の挙動は[04](./04-game-rules.md)の失点条件に従う（空振り＝ボールが自コートを通過した時に失点、ボタン連打自体では失点しない）。
- **SHOULD**: ボールが近い時にAボタンをハイライトしてHITタイミングを示唆してよい。
- 入力強度: タップ=通常ショット、ボールに勢いがある/高速接近時=スマッシュ（速度上昇）。判定は[04](./04-game-rules.md) ショット物理に従う。

### 3.3.2 コースの打ち分け（A応用）

- **MUST**: ボールを打つ瞬間の左ジョイスティックの傾きでコースを決める。
  - 右に倒している → 自分から見て**右方向**へ。
  - 左に倒している → 自分から見て**左方向**へ。
  - 中立 → ストレート寄り。
- **MUST**: 「自分から見て」の方向は[02](./02-coordinate-and-camera.md)の反転規則で論理化する（クライアントの右=論理の左）。
- 角度量は傾き量に比例させてよい（SHOULD）。上限角は[04](./04-game-rules.md)。

### 3.3.3 Bボタン：ロブショット

- **MUST**: タップで、相手の頭上を抜く高い山なりの軌道（低速・大きな滞空・浅く深く落とす）でボールを返す。
- 2Dトップダウンのため「高さ」は擬似表現する（MUST）: ボールの`height`値（論理高さ）を持ち、影/拡大縮小で高さを可視化（[06](./06-data-models.md)）。
- **MUST**: ロブ中のボールは相手のスマッシュ対象になり得る（頭上を越えられず手前で叩かれると不利）。打ち分けはAと同様に可能（SHOULD）。

## 3.4 オリエンテーション・初期案内

- **MUST**: 縦持ち検出時は半透明オーバーレイで「端末を横にしてください」を表示し、横向きで自動復帰する。
- **MUST**: 横画面固定は **CSS/オーバーレイ案内方式** で実現する。`screen.orientation.lock()` は **iOS Safari で非対応**のため、これに依存しない（[09](./09-open-questions.md) Q-9）。Android Chrome では `lock('landscape')` を併用してよい（MAY）。
- **MUST**: サウンド（Q-7の最小限効果音）は **最初のユーザー操作（タップ）で AudioContext を解放**してから再生する。iOS Safari の自動再生制限への対応（[09](./09-open-questions.md) Q-7/Q-9）。
- **SHOULD**: 初回起動時に操作チュートリアル（左で移動・右でショット）を1画面表示。スキップ可能。
- **SHOULD**: 端末のスリープ防止（Screen Wake Lock API）を試行する（**iOS含め非対応端末はフォールバック必須**）。
- **SHOULD**: 起動時に最低要件（iOS Safari 15+ / Android Chrome 90+）を満たさない場合は非対応メッセージを表示する。
- **MAY**: ハプティクス（バイブ）でヒット時のフィードバックを与えてよい（iOSは制限あり）。

## 3.5 アクセシビリティ/操作品質

- **MUST**: タッチ遅延を最小化（`touchstart`基準、`click`の300ms遅延を回避）。
- **SHOULD**: 指サイズを考慮し、ボタン最小ターゲット 48dp 以上。
- **SHOULD**: 利き手に依らず使えるよう左右レイアウトは固定（移動=左、ショット=右）だが、設定で左右反転を許可してもよい（MAY）。

## 3.6 検証観点（→ [07](./07-acceptance-criteria.md)）

- 左半分の任意位置タッチでジョイスティックが出現し、離すと消える。
- 左右同時操作（移動しながらショット）が成立する。
- 打つ瞬間のジョイスティック傾きでボールのコースが左右に変わる。
