# 04. ゲームルール・マッチング・試合設定

## 4.1 ゲームモード（マッチング形式）

| モード | 人数 | 接続 | 説明 |
|---|---|---|---|
| 壁打ち（1人） | 1 | P2P不要（ローカル完結） | 相手側が自動で跳ね返す「壁」。ラリー練習・通信テスト・フェーズ1の単体動作確認用。 |
| シングルス（1vs1） | 2 | ホスト⇔クライアント1本 | 基本対戦モード。 |
| ダブルス（2vs2） | 4 | スター型（ホスト⇔各クライアント） | 同一ルームに最大4人。2人1チーム。 |

要件:
- **MUST**: ルーム作成時にモードと定員を確定する（後から変更不可、変更は作り直し）。
- **MUST**: 定員に満たない/超過する参加は拒否し、理由を提示する。
- **MUST**: 壁打ちはオフラインでも起動できること（シグナリング/ネット接続不要）。

### 壁打ちモードの「壁」挙動
- **MUST**: 相手コートの所定ラインでボールを反射し、こちらへ返す。反射角・速度は調整可能な簡易ロジック（入射角反射 + ランダム微小ばらつき）。
- 得点は付けない、または「連続ラリー回数」をスコアとして表示してよい（SHOULD）。

## 4.2 試合設定（ミニマル）

休み時間に終わる長さを既定とする（G-4）。

| 設定項目 | 選択肢 | 既定 |
|---|---|---|
| 先取ポイント | 3 / 5 / 7 ポイント先取 | **3ポイント先取** |
| ゲーム数 | 1ゲーム先取（単ゲーム） | 1ゲーム |
| サーブ | ポイントごとに交互、または2本交代 | ポイントごと交互 |
| デュース | あり / なし | **なし**（先取数到達で即勝利） |

- **MUST**: ルーム作成画面で先取ポイント数を選べる（[08](./08-roadmap.md) フェーズ4）。
- **MUST**: 設定はホストが決定し、全クライアントに配信・固定される。
- **MAY**: 制限時間モード（n分で多くポイントを取った方が勝ち）を追加してよい。

## 4.3 ポイント進行・サーブ

- **MUST**: 各ポイントはサーブ（ボール投入）で開始する。サーブ側・サーブ位置はホストが決定し MATCH_STATE で配信する（[06](./06-data-models.md)）。
- サーブ操作はAボタン相当の入力で打ち出す（SHOULD）。簡易化として自動トスでもよい（MAY）。
- **MUST**: ポイント決着後、次サーブ前に短い演出/インターバルを挟む（得点表示、[04.5]）。

## 4.4 失点・得点条件

ボールの権限保持ピアが下記イベントを検知し、ホストへ報告 → ホストが POINT_RESULT を確定（[01](./01-architecture.md) 1.4）。

得点が相手に入る（自分の失点）条件:
1. **アウト**: ボールが自コートのコート外ライン外側で地面に落ちた／規定領域外へ出た。
2. **空振り/取り逃し**: ボールが自コートのベースライン側を通過し、有効に打ち返せなかった。
3. **ネット**: 自分の打球がネットを越えられなかった（`NET_Y`手前で失速・落下）。
4. **2バウンド**: ボールが自コート内で規定回数（既定1バウンド）を超えてバウンドした。

得点が自分に入る条件は上記の鏡（相手側で発生）。
- **MUST**: イン/アウト・ネット・バウンド判定は権限保持ピアが論理座標で計算する。
- **MUST**: 競合（双方が同時に主張、または食い違い）時はホストが裁定し、その結果を最終とする。
- **MUST**: 判定が曖昧/不整合な場合のホスト裁定方針（Q-4 確定）: 送信時刻 `t`（補正後）が早い報告を採用。判定不能時は**レット**として無得点でサーブやり直し（`rallyId`を進める）（[09](./09-open-questions.md) Q-4, [05](./05-netcode-sync.md) 5.4.3）。

## 4.5 スコア表示・演出・ゲームセット

- **MUST**: 各画面上部に「自分（自チーム）スコア / 相手スコア」を常時表示。各ピアは自分視点で表示する。
- **MUST**: 得点時に得点演出（短時間）。誰が得点したか分かること。
- **MUST**: 先取ポイント到達でゲームセット画面を表示（勝者/敗者、最終スコア、再戦/退出ボタン）。
- **SHOULD**: 再戦（Rematch）はホストが開始、設定を引き継ぐ。
- スコアの「正」はホストが保持し、POINT_RESULT/MATCH_STATE で全員へ同期する（MUST、[01](./01-architecture.md)）。

## 4.6 ショット物理（Arcade Physics 方針）

2Dトップダウン + 擬似高さ。Arcade Physicsで水平面（x,y）を扱い、高さ`h`は独自に積分する（MUST）。

| 要素 | 規定 |
|---|---|
| 通常ショット | 中速・低めの軌道。打ち分け角は最大 `SHOT_MAX_ANGLE`（例 ±35°）。 |
| スマッシュ | 高速・直線的。ボールが高め/高速接近時にAで発動。 |
| ロブ | 低速・`h`を大きく上げ滞空長い山なり。相手の頭上を狙う。 |
| バウンド | コート接地（`h<=0`）で反発、`h`方向に減衰反発。水平速度は摩擦で微減。 |
| ネット | `NET_Y`通過時に`h < NET_HEIGHT`なら通過失敗（ネットにかかる）。 |

実装定数（既定・調整可）:

| 定数 | 既定 | 説明 |
|---|---|---|
| `SHOT_SPEED` | 520 | 通常ショット速度（論理/秒） |
| `SMASH_SPEED` | 820 | スマッシュ速度 |
| `LOB_SPEED` | 300 | ロブ水平速度 |
| `SHOT_MAX_ANGLE` | 35° | 打ち分け最大角 |
| `HIT_RANGE` | 90 | ヒット可能半径（論理単位） |
| `NET_HEIGHT` | 1.0 | ネットを越えるのに必要な擬似高さ閾値 |
| `MAX_BOUNCE` | 1 | 自コートで許容するバウンド回数 |
| `SMASH_HEIGHT` | 0.6 | この高さ以上のボールをAで叩くとスマッシュ（[09](./09-open-questions.md) Q-6） |
| `SMASH_INCOMING_SPEED` | 600 | この接近速度以上をAで打つとスマッシュ |

### スマッシュ発動条件（Q-6 確定）
- **MUST**: Aヒット時に `ball.height >= SMASH_HEIGHT` または 接近速度 `>= SMASH_INCOMING_SPEED` を満たすとスマッシュ（`SMASH_SPEED`）。それ以外は通常ショット（`SHOT_SPEED`）。

### プレイヤー可動範囲（Q-5 確定）
- **MUST**: 各プレイヤーは自コート内のみ移動可能。ネット（`NET_Y`）を越えられない。横はサイドラインまで。可動矩形は `GameConfig` で定義する。

> これらは「ゲームバランス調整パラメータ」として1ファイルに集約すべき（SHOULD、[06](./06-data-models.md) Config）。
